# Infrastructure Details

Detailed reference for local development infrastructure.
For rules and quick reference, see `.claude/rules/infrastructure.md`.

## Local Development

All services run locally. Port isolation supports multiple repo instances in parallel.

### Port Configuration

Generated by `infrastructure/scripts/setup-ports.sh`:
- Backend: `8080 + REPO_INDEX`
- PostgreSQL: `5432 + REPO_INDEX`
- MailHog SMTP: `1025 + REPO_INDEX`
- MailHog HTTP: `8025 + REPO_INDEX`

### Service Scripts

| Script | Purpose |
|--------|---------|
| `run-backend.sh` | Build and start Go backend (`go run ./backend/application/`) |
| `run-infra.sh` | Start Docker infrastructure (MailHog, PostgreSQL) |
| `stop-backend.sh` | Stop backend by port PID |
| `stop-infra.sh` | Stop Docker infrastructure |
| `test-acceptance.sh` | Run acceptance tests with correct env vars |

### Network Flow (Local)

```
Client → backend (BACKEND_PORT)

Backend:
  → PostgreSQL (DB_PORT)
  → MailHog (MAILHOG_SMTP_PORT)
```
