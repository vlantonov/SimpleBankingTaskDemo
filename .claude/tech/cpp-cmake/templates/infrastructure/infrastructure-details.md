# Infrastructure Details -- C++/CMake

Detailed reference for local development infrastructure.
For rules and quick reference, see `.claude/rules/infrastructure.md`.

## Local Development

All services run locally. Port isolation supports multiple repo instances in parallel.

### Port Configuration

Generated by `infrastructure/scripts/setup-ports.sh`:
- Backend (gRPC): `50051 + REPO_INDEX`
- MailHog SMTP: `1025 + REPO_INDEX`
- MailHog HTTP: `8025 + REPO_INDEX`

### Build Commands

```bash
# Configure
cmake -B build -DCMAKE_BUILD_TYPE=Debug -DENABLE_COVERAGE=ON

# Build all
cmake --build build --parallel

# Build specific module
cmake --build build --target usecase_tests
cmake --build build --target acceptance_tests
```

### Service Scripts

| Script | Purpose |
|--------|---------|
| `run-backend.sh` | Build and start backend server |
| `run-infra.sh` | Start Docker infrastructure (MailHog) |
| `stop-backend.sh` | Stop backend by port PID |
| `stop-infra.sh` | Stop Docker infrastructure |
| `test-acceptance.sh` | Run acceptance tests with correct env vars |

### Network Flow (Local)

```
Client → backend gRPC server (BACKEND_PORT)

Backend:
  → MailHog (MAILHOG_SMTP_PORT)
  → SQLite (file-based, no network)
```
